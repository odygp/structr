# Adding a New Variant to an Existing Section Category

This document is a self-contained guide for adding a new **variant** to an existing section category in the Structr wireframe builder. A variant is a different **layout** of the same content fields. For example, `hero-centered` and `hero-split` both use title/subtitle/CTA fields but arrange them differently.

Adding a variant is simpler than adding a new category -- fewer files to touch, no schema changes, and no AI prompt updates needed.

## 1. Overview: What Is a Variant?

A **variant** reuses the parent category's `contentSchema` (the editable fields) but arranges them in a visually distinct layout. All variants within a category share the same schema -- users can switch between variants without losing content.

**Example -- Hero category variants:**
| Variant ID | Layout |
|---|---|
| `hero-centered` | Title + subtitle + buttons centered in a single column |
| `hero-split` | Title + subtitle + buttons on the left, image placeholder on the right |
| `hero-with-image` | Centered text above a large image placeholder |
| `hero-minimal` | Just a title + single CTA, no subtitle |
| `hero-with-form` | Title + subtitle on the left, signup form on the right |

All five share the same `contentSchema`: `title`, `subtitle`, `showPrimaryButton`, `ctaText`, `showSecondaryButton`, `ctaSecondaryText`, `showTertiaryButton`, `ctaTertiaryText`.

**Files you will touch (only 3):**
1. `src/components/sections/{category}/{VariantName}.tsx` -- NEW file (the component)
2. `src/lib/registry.ts` -- ADD entry to existing `variants` array
3. `src/lib/export-html.ts` -- ADD HTML export renderer function

**Files auto-generated (do not hand-edit):**
- `src/components/sections/index.ts` -- regenerated by `npm run generate:sections`

**Files you do NOT touch:**
- `src/lib/types.ts` -- no changes (category type already exists)
- `src/lib/ai/system-prompt.ts` -- no changes (registry auto-exposes variants to AI)

## 4. Step 2 -- Create Variant Component TSX

Create a new file at:
```
src/components/sections/{category}/{VariantName}.tsx
```

### Naming Convention

The file name must be PascalCase. The script `generate-section-index.mjs` converts it to a kebab-case `variantId` using this algorithm:
- Split PascalCase into parts
- Lowercase each part
- Replace the first part (component prefix) with the actual category directory name
- Join with hyphens

**Examples:**
| File Name | Category Dir | Generated variantId |
|---|---|---|
| `HeroCentered.tsx` | `hero/` | `hero-centered` |
| `FeaturesGrid.tsx` | `features/` | `features-grid` |
| `Pricing3Col.tsx` | `pricing/` | `pricing-3col` |
| `CtaWithImage.tsx` | `cta/` | `cta-with-image` |
| `FooterMinimal.tsx` | `footer/` | `footer-minimal` |
| `StoreWithFilters.tsx` | `store/` | `store-with-filters` |

### Component Template

Every section component follows this exact structure:

```tsx
'use client';
import { ColorMode } from '@/lib/types';
import { getColors } from '@/lib/colors';
import { getSpacingClasses } from '@/lib/spacing';
import EditableText from '@/components/builder/EditableText';

export default function VariantName({
  content,
  colorMode,
  sectionId,
}: {
  content: Record<string, any>;
  colorMode?: ColorMode;
  sectionId?: string;
}) {
  const c = getColors(colorMode || 'light');
  const spacing = getSpacingClasses(content._spacing as string, 'CATEGORY_NAME');
  const id = sectionId || '';

  return (
    <section className={`${c.bg} ${spacing}`}>
      <div className="max-w-7xl mx-auto">
        {/* Your layout here using content fields + color tokens */}
      </div>
    </section>
  );
}
```

### Required Imports

| Import | Purpose | When to use |
|---|---|---|
| `getColors` from `@/lib/colors` | Color tokens (bg, text, border, button classes) | Always |
| `getSpacingClasses` from `@/lib/spacing` | Section padding based on `_spacing` prop | Always |
| `EditableText` from `@/components/builder/EditableText` | Inline-editable text fields | For all text fields |
| `getGridColsClass` from `@/lib/columns` | Dynamic grid column classes | When grid columns are configurable |

### Color Token Usage (via `getColors()`)

The `c` object returned by `getColors(colorMode)` provides these Tailwind class strings:

```ts
// Backgrounds
c.bg            // Default background (white / gray-900)
c.bgAlt         // Alternate background (gray-50 / gray-800)
c.bgMuted       // Muted background (gray-100 / gray-700)
c.bgCard        // Card background (white / gray-800)
c.bgCardAlt     // Alt card background (gray-50 / gray-700)
c.bgPlaceholder // Placeholder/image background (gray-200 / gray-700)
c.bgAvatar      // Avatar background (gray-300 / gray-600)

// Text
c.text          // Primary text (gray-900 / white)
c.textSecondary // Secondary text (gray-600 / gray-300)
c.textMuted     // Muted text (gray-500 / gray-400)
c.textLight     // Faint text (gray-400 / gray-500)

// Buttons (compound classes -- apply directly)
c.btnPrimary    // Primary button: dark bg + light text
c.btnSecondary  // Secondary button: border + muted text
c.btnSmall      // Same as btnPrimary (alias)

// Borders
c.border        // Default border (gray-200 / gray-700)
c.borderLight   // Light border (gray-100 / gray-800)
c.divider       // Divider utility (divide-gray-200 / divide-gray-700)

// Form input (compound class)
c.input         // Input: bg + border + text + placeholder colors

// Highlighted card (e.g., featured pricing)
c.hlBg          // Highlighted bg (gray-900 / white)
c.hlText        // Highlighted text (white / gray-900)
c.hlTextSecondary // Highlighted secondary text
c.hlBtn         // Highlighted button
```

### Spacing Classes

`getSpacingClasses(content._spacing, 'category')` returns padding classes. The category name affects the preset:

| Category | Default | Compact | Spacious |
|---|---|---|---|
| `header`, `banner` | `py-4 px-4 @md:px-8` | `py-2 px-3 @md:px-4` | `py-6 px-4 @md:px-12` |
| `footer` | `py-10 @md:py-16 px-4 @md:px-8` | `py-6 @md:py-8 px-4 @md:px-6` | `py-14 @md:py-20 px-4 @md:px-12` |
| Everything else | `py-14 @md:py-20 px-4 @md:px-8` | `py-8 @md:py-10 px-4 @md:px-6` | `py-20 @md:py-28 px-4 @md:px-12` |

### EditableText Usage

**CRITICAL:** All user-visible text MUST use `EditableText` so it is inline-editable in the builder.

**Simple text field:**
```tsx
<EditableText
  sectionId={id}
  fieldKey="title"
  value={content.title as string}
  placeholder="Add headline..."
  tag="h1"
  className={`text-3xl @md:text-5xl font-bold ${c.text} leading-tight`}
/>
```

**Text inside a wrapper element (e.g., button):**
```tsx
<button className={`${c.btnPrimary} rounded-lg px-6 py-3 text-sm font-medium`}>
  <EditableText
    sectionId={id}
    fieldKey="ctaText"
    value={content.ctaText as string}
    placeholder="Button"
  />
</button>
```

**Items array field (e.g., feature list):**
```tsx
{features.map((feature, i) => (
  <div key={i}>
    <EditableText
      sectionId={id}
      fieldKey="features"
      arrayKey="features"
      itemIndex={i}
      itemFieldKey="title"
      value={feature.title}
      tag="h3"
      className={`text-lg font-semibold ${c.text}`}
    />
  </div>
))}
```

**EditableText props reference:**
| Prop | Type | Required | Description |
|---|---|---|---|
| `sectionId` | `string` | Yes | The section's unique ID |
| `fieldKey` | `string` | Yes | Content field key (e.g., `"title"`) |
| `value` | `string` | Yes | Current field value |
| `className` | `string` | No | Tailwind classes for styling |
| `placeholder` | `string` | No | Placeholder when empty (default: `"Click to edit..."`) |
| `tag` | `'h1' \| 'h2' \| 'h3' \| 'p' \| 'span' \| 'div'` | No | HTML tag to render (default: `'span'`) |
| `arrayKey` | `string` | No | For items arrays: the array field key |
| `itemIndex` | `number` | No | For items arrays: the item index |
| `itemFieldKey` | `string` | No | For items arrays: the field within the item |

### Full Example: Two Variants of the Same Category

**HeroCentered.tsx** -- centered single-column layout:
```tsx
'use client';
import { ColorMode } from '@/lib/types';
import { getColors } from '@/lib/colors';
import { getSpacingClasses } from '@/lib/spacing';
import EditableText from '@/components/builder/EditableText';

export default function HeroCentered({ content, colorMode, sectionId }: { content: Record<string, any>; colorMode?: ColorMode; sectionId?: string }) {
  const c = getColors(colorMode || 'light');
  const spacing = getSpacingClasses(content._spacing as string, 'hero');
  const id = sectionId || '';

  return (
    <section className={`${c.bgAlt} ${spacing}`}>
      <div className="max-w-3xl mx-auto text-center">
        <EditableText
          sectionId={id}
          fieldKey="title"
          value={content.title as string}
          placeholder="Add headline..."
          tag="h1"
          className={`text-3xl @md:text-5xl font-bold ${c.text} leading-tight`}
        />
        <EditableText
          sectionId={id}
          fieldKey="subtitle"
          value={content.subtitle as string}
          placeholder="Add subtitle..."
          tag="p"
          className={`mt-4 @md:mt-6 text-lg @md:text-xl ${c.textSecondary} leading-relaxed`}
        />
        {(content.showPrimaryButton || content.showSecondaryButton || content.showTertiaryButton) && (
          <div className="mt-10 flex flex-col @sm:flex-row items-center justify-center gap-3 @sm:gap-4">
            {content.showPrimaryButton !== false && content.ctaText && (
              <button className={`${c.btnPrimary} rounded-lg px-6 py-3 text-sm font-medium`}>
                <EditableText sectionId={id} fieldKey="ctaText" value={content.ctaText as string} placeholder="Button" />
              </button>
            )}
            {content.showSecondaryButton && content.ctaSecondaryText && (
              <button className={`${c.btnSecondary} rounded-lg px-6 py-3 text-sm font-medium`}>
                <EditableText sectionId={id} fieldKey="ctaSecondaryText" value={content.ctaSecondaryText as string} placeholder="Button" />
              </button>
            )}
            {content.showTertiaryButton && content.ctaTertiaryText && (
              <span className={`text-sm font-medium ${c.textSecondary} underline underline-offset-4 cursor-pointer`}>
                <EditableText sectionId={id} fieldKey="ctaTertiaryText" value={content.ctaTertiaryText as string} placeholder="Link text" />
              </span>
            )}
          </div>
        )}
      </div>
    </section>
  );
}
```

## 5. Step 3 -- Add to Registry Variants Array

Open `src/lib/registry.ts`. Find the parent category block and add a new entry to its `variants` array.

### Format

```ts
{ variantId: 'CATEGORY-VARIANT_SLUG', variantName: 'Human Label', defaultContent: { ...defaultCategoryContent, /* any overrides */ } },
```

### Rules

1. **`variantId`** must match the kebab-case ID that `generate-section-index.mjs` will produce from your file name (see naming convention table in Step 2).
2. **`variantName`** is what users see in the variant picker UI. Keep it short (2-4 words).
3. **`defaultContent`** must include ALL fields from the `contentSchema`. Spread the existing `defaultXxxContent` constant and override as needed.

### Example

Adding a `hero-video` variant:
```ts
// Find the hero category block:
variants: [
  { variantId: 'hero-centered', variantName: 'Centered', defaultContent: { ... } },
  { variantId: 'hero-split', variantName: 'Split with Image', defaultContent: { ... } },
  // ... existing variants ...
  // ADD YOUR NEW VARIANT HERE:
  { variantId: 'hero-video', variantName: 'With Video', defaultContent: { ...defaultHeroContent, showPrimaryButton: true, showSecondaryButton: true, showTertiaryButton: false, ctaTertiaryText: '' } },
],
```

| `defaultHeroContent` | `hero` |

| `defaultPricingPlans` | `pricing` (array) |

## 6. Step 4 -- Add HTML Export Renderer

Open `src/lib/export-html.ts`. Add a new entry to the `htmlRenderers` record.

### Format

The renderer is a function that takes `SectionContent` and returns an HTML string. The HTML should be a standalone, self-contained section using Tailwind utility classes.

```ts
const htmlRenderers: Record<string, HtmlRenderer> = {
  // ... existing renderers ...
  'variant-id': (c) => `<section class="py-24 px-8">
  <div class="max-w-7xl mx-auto">
    <h1 class="text-5xl font-bold text-gray-900 mb-6">${escapeHtml(str(c.title))}</h1>
    <p class="text-xl text-gray-600 mb-8">${escapeHtml(str(c.subtitle))}</p>
  </div>
</section>`,
};
```

### Helper Functions Available

| Function | Usage |
|---|---|
| `escapeHtml(string)` | Escape HTML entities in user content. **Always use this.** |
| `str(value)` | Safely convert any value to string (returns `''` if not a string). |
| `renderItems(items, fallback)` | Safely extract items array with optional fallback. Returns `ContentItem[]`. |

### Important Rules

1. **Always wrap user content in `escapeHtml(str(...))`** -- never insert raw content.
2. **Use hardcoded Tailwind classes** (not token variables) -- the HTML export is standalone.
3. **Use gray color palette** for light mode: `text-gray-900`, `text-gray-600`, `bg-gray-50`, etc.

5. **Match the visual layout** of your TSX component as closely as possible.

## 10. Responsiveness Rules

### Container Queries, NOT Viewport

This project uses **container queries** (`@container`) instead of viewport media queries. All responsive prefixes use `@` notation:

| Prefix | Container Width |
|---|---|
| `@sm:` | >= 384px |
| `@md:` | >= 640px |
| `@lg:` | >= 768px |
| `@xl:` | >= 1024px |

**NEVER use** standard viewport prefixes (`sm:`, `md:`, `lg:`, `xl:`). Always use `@sm:`, `@md:`, `@lg:`, `@xl:`.

### Standard Responsive Patterns

**Typography scaling:**
```tsx
className="text-3xl @md:text-5xl font-bold"   // Headlines
className="text-lg @md:text-xl"                // Subtitles
className="text-sm @md:text-base"              // Body text
```

**Grid layouts:**
```tsx
className="grid grid-cols-1 @md:grid-cols-2 gap-8"      // 2-column
className="grid grid-cols-1 @md:grid-cols-3 gap-8"      // 3-column
className="grid grid-cols-2 @md:grid-cols-4 gap-8"      // 4-column
className="grid grid-cols-1 @lg:grid-cols-2 gap-12"     // Split layout
```

**Flex direction:**
```tsx
className="flex flex-col @sm:flex-row gap-4"   // Stack on mobile, row on wider
```

**Spacing scaling:**
```tsx
className="mt-4 @md:mt-6"   // Tighter on mobile
className="gap-4 @md:gap-8" // Smaller gaps on mobile
```

### Array Field Handling

When rendering items arrays, always provide a fallback:
```tsx
const features = (content.features as Array<{title: string; description: string}>) || [];
```

Use `.map()` with `key={i}` for rendering:
```tsx
{features.map((feature, i) => (
  <div key={i}>...</div>
))}
```
